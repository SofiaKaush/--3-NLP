Полное руководство по развертыванию AI-ботов

Данный репозиторий содержит серию веб-приложений (чат-ботов) на базе Flask, которые демонстрируют интеграцию с OpenAI-совместимыми API (в частности, OpenRouter).

В проекте представлены 4 версии приложения:

app_v1.py — Базовый текстовый чат-бот.

app_v2.py — Мультимодальный бот (поддержка текста и загрузки изображений).

app_v3.py — Чат-бот с явной генерацией изображений (по кнопке).

app_v4.py — Умный агент (Function Calling), который сам решает, когда рисовать картинки, а когда показывать погоду.

🛠 Структура репозитория

Для успешного запуска убедитесь, что в вашей папке находятся следующие файлы:

💻 Способ 1: Локальный запуск (для разработки)

Если вы хотите запустить и протестировать скрипты прямо на своем компьютере:

1. Установка Python и окружения

Убедитесь, что у вас установлен Python (рекомендуется версия 3.10+). Откройте терминал в папке проекта и создайте виртуальное окружение:

Для Windows:

python -m venv venv
venv\Scripts\activate


Для macOS/Linux:

python3 -m venv venv
source venv/bin/activate


2. Установка зависимостей

Чтобы избежать ошибок с отсутствующими модулями, установите необходимые библиотеки напрямую:

pip install flask openai


(Примечание: Если на macOS/Linux команда выдает ошибку, используйте pip3 install flask openai)

3. Настройка API ключа

Приложения используют бесплатные модели через OpenRouter. Вам необходимо получить API-ключ на сайте OpenRouter.ai и добавить его в переменные окружения.

Для Windows (cmd):

set OPENROUTER_API_KEY=ваш_ключ_здесь


Для macOS/Linux:

export OPENROUTER_API_KEY="ваш_ключ_здесь"


4. Запуск приложения

Выберите нужную версию скрипта и запустите её. Например, для запуска умного агента:

python app_v4.py


После запуска откройте браузер и перейдите по адресу: http://localhost:5004 (порт зависит от версии скрипта, он выводится в терминале).

🐳 Способ 2: Запуск через Docker

Docker позволяет запустить приложение в изолированном контейнере, где уже настроено всё необходимое.

Убедитесь, что у вас установлен и запущен Docker Desktop.

Соберите Docker-образ, находясь в папке с проектом:

docker build -t ai-flask-bot .


Запустите контейнер, пробросив порт 5004 и передав API ключ:

docker run -p 5004:5004 -e OPENROUTER_API_KEY="ваш_ключ_здесь" ai-flask-bot


Приложение будет доступно по адресу http://localhost:5004.

🐙 Способ 3: Сохранение всех файлов на GitHub

Если вы хотите сохранить исходный код проекта на GitHub:

Шаг 1. Инициализация Git

Откройте терминал в папке проекта и выполните команды для создания локального репозитория:

Зайдите на свой аккаунт GitHub и создайте новый репозиторий (при создании не добавляйте файлы README, .gitignore или license, чтобы репозиторий был пустым).

git branch -M main

🚀 Способ 4: Развертывание в GitLab с CI/CD

В проекте уже настроен файл .gitlab-ci.yml, который автоматически проверяет код на ошибки (Lint) и собирает Docker-образ (Build) при каждом обновлении ветки main.

Шаг 1. Инициализация репозитория

Откройте терминал в папке проекта и выполните команды:

git init
git add .
git commit -m "Initial commit"


Шаг 2. Загрузка в GitLab

Создайте пустой проект в вашем аккаунте GitLab. Затем свяжите локальный репозиторий с GitLab и отправьте файлы:

git remote add origin [https://gitlab.com/ВАШ_USERNAME/ИМЯ_ПРОЕКТА.git](https://gitlab.com/ВАШ_USERNAME/ИМЯ_ПРОЕКТА.git)
git branch -M main
git push -u origin main


Шаг 3. Настройка секретов (API ключа) в GitLab

Чтобы ваше приложение и контейнер могли обращаться к нейросети, нужно безопасно передать ключ в GitLab:

Зайдите в ваш проект на GitLab.

Перейдите в Settings (Настройки) -> CI/CD.

Найдите раздел Variables (Переменные) и нажмите Expand (Развернуть).

Нажмите Add variable (Добавить переменную):

Key: OPENROUTER_API_KEY

Value: вставьте ваш реальный ключ от OpenRouter

Убедитесь, что галочка Mask variable включена (это скроет ключ из логов).

Сохраните.

Шаг 4. Проверка пайплайна

Перейдите во вкладку Build -> Pipelines. Вы увидите, как GitLab выполняет стадии lint (проверка кода) и build (сборка Docker-образа).
После успешной сборки ваш Docker-образ будет сохранен во встроенном реестре Deploy -> Container Registry, откуда вы сможете вытянуть его на любой сервер!
